# 線上讀書會文章新增規則

此文件記錄了每週新增文章的標準作業程序 (SOP)。

## 1. 文章檔案規範
- **檔案命名**：使用當天日期 `yyyyMMdd.html` 格式。
  - 例如：`20260324.html`。
  - 若當天有第二篇，則命名為 `20260324-1.html`，依此類推。
- **檔案內容碼**：
  - 必須包含 `<head>` 引入 `content.css` 與 `bookmark.js`。
  - 必須包含書籤按鈕：`<button onclick="bookmark()"><img src="bookmark.png"/>書籤</button>`。

## 2. 文章分段
  - 依文意分行，如果是對話，需換至下一行開始。
  範例:
  ```
  <h1>《孟子》盡心章句上．第三十六章</h1>
  <h1>孟子自范之齊，望見齊王之子。<ruby>喟<rt>ㄎㄨㄟˋ</rt></ruby>然嘆曰：</h1>
  <h1>　「居移氣，養移體，大哉居乎！夫非盡人之子<ruby>與<rt>ㄩˊ</rt></ruby>？」</h1>
  <h1> 孟子曰：</h1>
  <h1>　「王子宮室、車馬、衣服多與人同；</h1>
  <h1>　　而王子若彼者，其居使之然也；</h1>
  <h1>　　況居天下之廣居者乎？</h1>
  <h1>　　魯君之宋，呼於<ruby>垤<rt>ㄉㄧㄝˊ</rt></ruby>澤之門。</h1>
  <h1>　　守者曰：『此非吾君也，何其聲之似我君也？』</h1>
  <h1>　　此無他，居相似也。」</h1>
  ```

## 3. 特殊讀音 (Ruby)
  - 在「原文」中使用 HTML `<ruby>` 語法標記特殊讀音或古音。
  - 格式：`<ruby>字<rt>注音</rt></ruby>`。
  - 例如：`<ruby>惡<rt>ㄨ</rt></ruby>`。

## 4. 單字/單詞註釋
  - 使用 `<h2>` 標籤記錄註釋。
  - 格式：`<h2>單字、詞：解釋</h2>`。

## 5. AI生成的註釋
  - 使用 `<h3>` 標籤記錄註釋。
  - 標題：`<h3>《AI生成的註釋》</h3>`。
  - 格式：`<h3>單字、詞：解釋</h3>`。

## 6. 內容正確性
  - **禁止任意修改**：必須嚴格依照使用者提供的原文內容進行排版，不可擅自變更文字或修正所謂的「錯字」。
  - **排版標記**：僅能新增 HTML 標籤（如 `<h1>`、`<ruby>`、`<h2>`）進行格式化。

## 7. 目錄更新 (Menu)
  - **menu.html**：按日期排序，在 `</body>` 前新增連結。
  - **menu-c.html**：按分類排序，在對應的書籍（如《孟子》）下方新增連結。
